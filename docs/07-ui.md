# UI Systems — Complete Guide

## SECTION 1: What Is This & Why Do You Need It?

### What It Does
The **UI system** is everything the player sees on screen that is not the 3D world: menus, health bars, dialogue boxes, the inventory grid, the skill bar. UI in Godot is made from Control nodes arranged in a tree, and sits on top of the 3D world on a separate canvas layer.

### What UI Is Used For

| UI Element | What It Is | Where in This Project |
|------------|-----------|----------------------|
| Dialogue box | Text panel that appears when talking to NPCs | `ui/dialogue/dialogueUI.tscn` |
| Inventory | 3×3 item grid (bottom-right) | `ui/inventory/inventoryUI.tscn` |
| Skill bar | 5-slot ability bar (bottom-center) | `ui/skill/skillUI.tscn` |
| Debug HUD | Position/velocity display (top-left) | `ui/hud/hud.tscn` |
| Pause menu | ESC overlay | `ui/pause_menu/` |
| Main menu | Title screen | `ui/main_menu/` |

### Why This Exists
UI in Godot is separate from 3D physics objects because it always needs to stay on screen regardless of camera movement. A health bar should always be at the top-left, not floating in 3D space.

## SECTION 2: Complete Guide (0 to 100%)

### Part A: How UI Works in Godot

#### The Canvas Layer
UI nodes live in a `CanvasLayer` node. This layer renders on top of the 3D world, completely separate from the camera. Even if the camera rotates, UI stays in place.

```
World (Node3D)              ← 3D scene
└── Player
    ├── CharacterBody3D
    └── HUD (CanvasLayer)   ← UI lives here, not in 3D
        ├── InventoryUI
        └── SkillUI
```

#### Control Nodes
All UI is made from `Control` nodes:
- `Control` — base node, invisible, defines a rectangle on screen
- `Panel` / `PanelContainer` — a box with a background
- `Label` — static text
- `RichTextLabel` — formatted text (bold, color, etc.)
- `Button` — clickable button
- `VBoxContainer` — stacks children vertically
- `HBoxContainer` — stacks children horizontally
- `GridContainer` — arranges children in a grid

#### Anchors and Offsets
Every `Control` has anchors that define where it sits on screen:
- Anchors go from 0.0 (left/top) to 1.0 (right/bottom)
- `anchor_left = 0.5, anchor_right = 0.5` → horizontally centered
- `anchor_top = 1.0, anchor_bottom = 1.0` → pinned to bottom
- Offsets then move it by pixels from that anchor position

### Part B: Create a Simple UI Panel

This creates a "testPanel" with a title and two buttons.

#### Step 1: Create the Scene

1. In the file browser, go to `ui/` (or create a subfolder `ui/testPanel/`)
2. Right-click → "New Scene"
3. Root node type: **Control**
4. Name it: `TestPanel`
5. Save as: `ui/testPanel/testPanel.tscn`

#### Step 2: Build the Node Tree

In the scene tree, add children to `TestPanel`:

```
TestPanel (Control)
└── Panel (PanelContainer)
    └── VBoxContainer
        ├── TitleLabel (Label)
        ├── Button1 (Button)
        └── Button2 (Button)
```

To add each node:
1. Right-click parent → "Create Child Node"
2. Search by type name and create

#### Step 3: Set Properties

Select `TestPanel` (root Control):
- In Inspector → **Layout** → **Anchors Preset** → choose "Center" (preset 5)

Select `TitleLabel`:
- Inspector → Text → type: `"Test Panel"`

Select `Button1`:
- Inspector → Text → type: `"Do Thing A"`

Select `Button2`:
- Inspector → Text → type: `"Do Thing B"`

#### Step 4: Create the Script

1. Right-click `TestPanel` → "Attach Script"
2. Save as: `ui/testPanel/testPanel.gd`
3. Paste:

```gdscript
extends Control

@onready var _button1: Button = $Panel/VBoxContainer/Button1
@onready var _button2: Button = $Panel/VBoxContainer/Button2

signal button1_pressed
signal button2_pressed

func _ready() -> void:
	_button1.pressed.connect(_on_button1)
	_button2.pressed.connect(_on_button2)

func _on_button1() -> void:
	button1_pressed.emit()
	print("Button 1 pressed")

func _on_button2() -> void:
	button2_pressed.emit()
	print("Button 2 pressed")

func show_panel() -> void:
	show()

func hide_panel() -> void:
	hide()
```

#### Step 5: Add to the Level

To show this panel in the game, open `levels/test/world.tscn` and drag `testPanel.tscn` into the scene tree under the UI CanvasLayer.

Or spawn it from a script:

```gdscript
var panel = load("res://ui/testPanel/testPanel.tscn").instantiate()
panel.button1_pressed.connect(_on_panel_button1)
add_child(panel)
```

### Part C: Common UI Patterns

#### Toggle Visibility

```gdscript
func toggle() -> void:
	visible = not visible
```

#### Show / Hide With Mouse Cursor

When UI appears, show the cursor. When it hides, recapture it:

```gdscript
func show_panel() -> void:
	show()
	Input.mouse_mode = Input.MOUSE_MODE_VISIBLE

func hide_panel() -> void:
	hide()
	Input.mouse_mode = Input.MOUSE_MODE_CAPTURED
```

#### Fade In / Fade Out Animation

```gdscript
func fade_in() -> void:
	modulate.a = 0
	show()
	var tween = create_tween()
	tween.tween_property(self, "modulate:a", 1.0, 0.3)

func fade_out() -> void:
	var tween = create_tween()
	tween.tween_property(self, "modulate:a", 0.0, 0.3)
	await tween.finished
	hide()
```

### Part D: Positioning UI Elements

#### Common Positions

| Where | Anchors Preset | Use For |
|-------|----------------|---------|
| Top-left | 0 | Debug info |
| Top-center | 1 | Title, notifications |
| Center | 5 | Modals, dialogue |
| Bottom-center | 7 | Health bar, skill bar |
| Bottom-right | 8 | Inventory |
| Full screen | 15 | Overlays, pause menu |

#### Example: Pin to Bottom-Right

Select the Control node, then in Inspector:
```
Layout → Anchors Preset → "Bottom Right" (preset 8)
```

Then use offsets to move it inward from the corner:
```
offset_left = -230
offset_top  = -230
offset_right = -8
offset_bottom = -8
```

### Part E: Layering (Which UI is on Top)

Use `CanvasLayer` to control draw order. Higher layer number = drawn on top:

```
layer = 5   → Background UI (world-space labels)
layer = 10  → HUD (inventory, skill bar)
layer = 20  → Menus (pause, settings)
layer = 30  → Overlay (loading screen, fade)
```

To create a CanvasLayer in a scene:
1. Add a `CanvasLayer` node
2. Set its `layer` property in Inspector

## SECTION 3: Quick Reference

### Node Types

| Node | Use |
|------|-----|
| `Control` | Base UI node |
| `PanelContainer` | Box with background panel |
| `Label` | Plain text |
| `RichTextLabel` | Text with BBcode formatting |
| `Button` | Clickable button |
| `VBoxContainer` | Stack children vertically |
| `HBoxContainer` | Stack children horizontally |
| `GridContainer` | Grid of items (set `columns`) |

### Basic Panel Script Template

```gdscript
extends Control

signal panel_closed

func _ready() -> void:
	# Connect button signals here

func show_panel() -> void:
	show()
	Input.mouse_mode = Input.MOUSE_MODE_VISIBLE

func hide_panel() -> void:
	hide()
	Input.mouse_mode = Input.MOUSE_MODE_CAPTURED
	panel_closed.emit()
```

### Mouse Modes

```gdscript
Input.mouse_mode = Input.MOUSE_MODE_VISIBLE    # Cursor visible, can click UI
Input.mouse_mode = Input.MOUSE_MODE_CAPTURED   # Cursor hidden, controls camera
```

## Troubleshooting

**UI appears behind the 3D world:**
- Put UI in a `CanvasLayer` node (layer 10+)
- Without a CanvasLayer, Control nodes can render behind 3D geometry

**Buttons are not clickable:**
- Check the root Control's `mouse_filter` property — it should not be "Ignore"
- Make sure mouse mode is `MOUSE_MODE_VISIBLE`

**Text is cut off:**
- Set the `Label`'s `autowrap_mode` to "Word" or "WordBreak" in Inspector

**Panel doesn't show on screen:**
- Check `visible = true` in Inspector
- Check anchors are set correctly (wrong anchors = panel off-screen)

**Inventory/skill UI not visible after adding to scene:**
- Check the slot `visible` property — Godot sometimes sets `visible = false` automatically
- Select each child node and verify `visible = true` in Inspector
